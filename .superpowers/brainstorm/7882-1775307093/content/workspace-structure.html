Inside Each Market Tab — Workspace Structure

How should each market's workspace be organized?

A

Tailored views per market

Each market gets sub-views that make sense for its domain. Equities keeps Heatmap/List/Race/Portfolio. Bonds gets Yield Curve / Credit Matrix / Duration Ladder. FX gets Rate Matrix / Heatmap / Carry Map. Real Estate gets Regional Map / Price Index / REIT screen. Each market feels purpose-built for analysts of that asset class.

📈 Equities 🏛️ Bonds 💱 FX 📊 Deriv 🏠 Real Estate 🛡️ Insurance
Heatmap List Bar Race Portfolio Radar ML Explorer
↑ equities sub-views (tailored)
Yield Curve Credit Matrix Duration Ladder Sovereign Spread
↑ bonds sub-views (tailored)
B

Consistent views across all markets

Every market gets the same 3 views: a Heatmap (treemap by size), a List (sortable table), and a Chart (time series). The data inside changes but the UI pattern is identical. Simpler to build and learn — but bonds and real estate don't naturally fit a treemap.

📈 Equities 🏛️ Bonds 💱 FX 📊 Deriv 🏠 Real Estate 🛡️ Insurance
Heatmap List Chart
same 3 views for ALL markets
Recommendation

A (tailored) — a yield curve for bonds is fundamentally different from a stock heatmap. Forcing the same layout onto all markets would produce mediocre analysis tools. Each market deserves its best-fit visualization.
